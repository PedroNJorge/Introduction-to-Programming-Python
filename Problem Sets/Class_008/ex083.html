[IP083] Three-Eyed Raven

Bran Stark, with his newfound abilities as the Three-Eyed Raven, needs to send a secure message to allies across the realm. To keep the message hidden from enemies, Bran decides to use the Vigenère cipher, a method of encrypting alphabetic text by using a keyword to shift letters in a repeating pattern. The Vigenère cipher is more secure than the simple Caesar cipher because it uses a series of shifts determined by the keyword.

Imagine for instance that text to cipher is "attackatdawn" and the keyword is "fog". Then:

  • The first letter 'a' of the text is shifted 5 positions to the right because the first letter 'f' of the keyword is the 5th letter of the alphabet (counting from zero).
  • The second letter 't' of the text is shifted 14 positions to the right because the second letter 'o' of the keyword is the 14th letter of the alphabet (counting from zero).
  • The third letter 't' of the text is shifted 6 positions to the right because the third letter 'g' of the keyword is the 6th letter of the alphabet (counting from zero).
  • The fourth letter 'a' of the text is shifted 5 positions to the right because the fourth letter 'f' of the keyword is the 5th letter of the alphabet (counting from zero). Note how the fourth letter wraps around the keyword, returning to the first letter.
  • (...)

The Problem

Write two functions:

  • vigenere_encrypt(text, keyword): that given two strings text and keyword returns a string that is the result of applying the Vigenère cipher to text using as key the keyword.
     
  • vigenere_decrypt(text, keyword): that given two strings text and keyword returns a string that is the result of applying the reverse of the Vigenère cipher to text using as key the keyword.
     

In both methods the keyword will always be a string made only of lowercase letters. As to what regards the text, it can have any type of character, but you should only shift the letter characteres. Note that the case of the letter (upper or lower case) should be kept in the encrypted/decrypted text and that the i-th letter of the text should always be shifted using the alphabet position of the i-th letter of the keyword.

Constraints

The following limits are guaranteed in all the test cases that will be given to your program:

1 ≤ |text| ≤ 200       length of the text
1 ≤ |keyword| ≤ 20       length of the keyword
